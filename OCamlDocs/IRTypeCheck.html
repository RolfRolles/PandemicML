Module IRTypeCheck


module IRTypeCheck: sig .. end
Support for interrogating the type of IR terms, and ensuring the semantic consistency of instructions.

Exceptions

exception TypeException of string
Raised by most of the functions in this module upon failure.

Type-checking functions

val typecheck_expr : IR.expr -> IR.var
Typecheck an expressions. Returns a variable "archetype", i.e., if we were to replace the expression with a variable, what would the type of the variable need to be? Raise a IRTypeCheck.TypeException upon failure.
val typecheck_instr : IR.instr -> unit
Typecheck an instruction. Returns nothing; a IRTypeCheck.TypeException is raised upon failure.

Type-related helper functions

val is_integer_type : IR.var -> bool
Is the variable an integer?
val is_memory_type : IR.var -> bool
Is the variable a memory?
val type_of_integer_type : IR.var -> IR.typereg
Returns the IR.typereg type of a given integer, or raises a IRTypeCheck.TypeException.
val size_of_integer_type : IR.var -> int
Returns the integer size (in bits) of a given integer, or raises a IRTypeCheck.TypeException.
val compare_types : IR.var -> IR.var -> bool
Memories match memories always (revisit for endianness?), integers match integers if their sizes match. Raise a IRTypeCheck.TypeException upon comparing an integer with a memory.
val endianness_of_memory_type : IR.var -> IR.typeendian
Returns the endianness of a given memory, or raises a IRTypeCheck.TypeException.
